# NOOP v9.0.2

A feature-and-accuracy release on top of 9.0.1. The headline is an opt-in **"optimal strain reached"** buzz, **experimental faster history sync**, your strap's real **pack voltage** in Devices, and a stack of accuracy fixes for workouts, steps and sleep. Everything still runs on your own device, offline, no account. Update from the Releases page, AltStore, or `brew upgrade`.

## New

- **"Optimal strain reached" notification (#593).** Opt in (it's off by default) and NOOP sends a single notification when the day's accumulated strain enters the optimal range for that day's recovery. It only fires for the day you're actively building — never for a backfilled or past day — and the copy is NOOP's own, not WHOOP's.
- **Strap pack voltage in Devices (#592).** Alongside the battery percent, Devices now shows the strap's measured pack voltage (WHOOP 4.0), read from the extended-battery response — a more honest read of remaining charge than the percentage alone. A read-only diagnostic probe (in Devices) can dump the full extended-battery frame for anyone helping map the remaining fields.
- **Faster history sync — experimental (#533).** New opt-in toggles ask the strap for a higher-priority connection and a faster radio mode (LE 2M PHY) for the duration of a history offload, so a deep backlog drains in fewer, quicker syncs. Off by default while it's proven out on more straps. The auto-continue cap was also raised so a large backlog finishes in one connection.
- **Recompute a sleep you deleted (#526).** Removing a bad sleep window no longer leaves a permanent hole — you can ask NOOP to recompute the night from the raw stream.

## Fixed

- **Second-strap workouts showed blank heart rate and calories (#512, #513).** Workout heart rate is now read under the strap that actually recorded the session, not a hardcoded/active device id, so workouts recorded on a second strap fill in correctly (iPhone, Mac, Android).
- **Today steps counted from the wrong source (#551, #574, #575).** Steps now route by their real source and blend an imported Apple Health count where present, matching Android; the order-dependent catalog reorder is gone.
- **Foot-sport step counts were half of reality (#568).** Activity-file (FIT) imports now double the reported cycle count into steps for running and walking sports, where one cycle is two steps.
- **Oura interval (IBI) imports decoded wrong (#511).** The 0x60/0x80 decoders now match the ring's real byte layout, so imported heartbeat intervals — and the HRV built from them — are correct.
- **The morning recap could fire twice or on the wrong day (#567).** Its once-per-recap gate is now keyed to the night it banked, not the calendar day.
- **Sleep time edits are saved atomically (#525),** so an interrupted edit can't leave a half-written night.
- **Pull-to-sync polish on Today (#582, #587, #590).** The pull indicator now shows "Syncing…", stays up for the real backfill instead of flickering per offload chunk, and hides at rest — it only appears when you pull.
- **iPhone now asks for notification permission during onboarding (#591),** so the alerts you enable can actually be delivered.
- **Stress marker values stay readable (#507),** and Android's battery dialog no longer contradicts Settings (#539).
- **WHOOP 5.0/MG config fix (#522):** the `enable_sig12` value is corrected to ASCII `'1'`.

Also: real plural forms and a batch of translation fixes across German, French, Spanish and Russian (#541–#544, #559), and WHOOP 5.0/MG decode groundwork (v20 optical records, #545/#546) continues under the hood.
